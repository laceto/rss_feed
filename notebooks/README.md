# Notebooks

Jupyter notebooks for exploring and consuming the financial news pipeline outputs.
All notebooks run with **no API key** unless noted. Constants are always at the top of each notebook.

---

## Start here

**`START_HERE.ipynb`**
Data availability check + routing table. Run this first to confirm all pipeline
outputs are present, then follow the link to the right notebook for your goal.

---

## Daily workflow (consume a pre-digested summary)

```
1. morning_briefing_guide   ← "What spiked today and why?"
2. query_sector_guide       ← "How is [sector] trending?"
3. query_entity_guide       ← "What's the read on [company]?"
```

---

## Notebook reference

### Insight notebooks (high-level, daily use)

| Notebook | Question it answers | API key? | Constants |
|---|---|---|---|
| `morning_briefing_guide.ipynb` | What spiked today, what was the narrative, what do the sectors say? | no | `DATE`, `TOP_N` |
| `emerging_topics_guide.ipynb` | What topics is the market suddenly talking about? | no | `DATE`, `TOP_N`, `LOOKBACK` |
| `query_sector_guide.ipynb` | What is the current and historical sentiment for a sector? | no | `SECTOR`, `LOOKBACK`, `FREQ` |
| `query_entity_guide.ipynb` | What is the sentiment trend for a company or person? | no | `ENTITY`, `LOOKBACK`, `TOP_N` |
| `semantic_search_guide.ipynb` | Ask a plain-English question about the news corpus | **yes** | `QUERY`, `STRATEGY`, `K_SEMANTIC`, `K_BM25` |

### Export & data notebooks

| Notebook | Question it answers | API key? | Constants |
|---|---|---|---|
| `signal_export_guide.ipynb` | Produce a backtest-ready signal file (sector + entity + spike flags) | no | `LOOKBACK`, `TOP_ENTITIES`, `OUTPUT_TSV` |
| `sector_db_guide.ipynb` | Query the raw SQLite database directly | no | `SECTOR`, `ENTITY`, `LOOKBACK_DAYS` |

## Data sources

| File | Generated by | Used by |
|---|---|---|
| `data/briefings/{date}.json` | `daily-briefing` CI workflow (weekdays ~13:00 UTC) | `morning_briefing_guide` |
| `data/sector_summary.tsv` | `collect-sector-results` CI workflow | all query notebooks |
| `data/topic_trends.tsv` | `collect-sector-results` CI workflow | `emerging_topics_guide`, `signal_export_guide` |
| `data/entity_sentiment_ts.tsv` | `collect-sector-results` CI workflow | `query_entity_guide`, `signal_export_guide` |
| FAISS vectorstore | `embed-feeds` CI workflow | `semantic_search_guide` |

If `sector_summary.tsv` is missing locally, the setup cell in `query_sector_guide.ipynb`
and `query_entity_guide.ipynb` will pull it automatically from HuggingFace (`lacetohf/sector-analysis`).

---

## Trend direction threshold

All notebooks use the same ±0.20 threshold (matching `pipeline/query_sector.py`):

- `> +0.20` → **improving**
- `< −0.20` → **deteriorating**
- otherwise → **stable**
